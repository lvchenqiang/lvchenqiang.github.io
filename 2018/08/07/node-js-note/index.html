初探Node.js

Node.js和javascript的区别

首先,node.js和js都基于ECMAScript运行。

ECMAScript

ECMAScript 作为基础提供了:

  • 定义语法,写node.js或者js都需要遵守
  • 变量定义、循环、判断、函数等

  • 原型和原型链、作用域和闭包、异步

  • 不能操作dom,不能监听click事件,不能发送ajax请求

  • 不能处理http请求,不能操作文件
  • 即,只有ECMAScript,几乎写不了实际的项目

javascript

  • 使用ECMAScript的语法规范,外加Web API(w3c定义),缺一不可。
  • DOM操作、BOM操作、事件绑定、Ajax等
  • 两者结合,完成浏览器前端的操作

数据验证

cookie
  • 存储在浏览器里面字符串(最大5kb)
  • 跨域不共享
  • 可以存储结构化数据
  • 每次发送http请求,会将请求域的cookie发送给server
  • server可以修改cookie,并返回给浏览器
  • 浏览器也可以通过js修改cookie(有限制)
session
  • 客户端存储userid,服务端查询对应的username
  • session 访问频繁,对性能要求极高
  • session可不考虑断电丢失数据的问题
  • session的数据量不会很大
redis
  • web server最常用的缓存数据库,数据存储在内存中
  • 相比于mysql,访问速度很快
  • 成本高,存储量小

安装教程:https://www.runoob.com/redis/redis-install.html

nginx配置
  • 高性能的web服务,开源免费
  • 一般用于静态服务、负载均衡
  • 反向代理
