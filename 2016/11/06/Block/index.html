Block 详解

发表于 2016-11-06 | 分类于 iOS ， BOOK |

Block 概要

Block 是C语言的扩充功能。用一句话表示这个扩充功能，就是带有自动变量的匿名函数。

匿名函数，就是所谓的不带有名称的函数。C语言的标准中不允许存在这样的函数，必须使用函数的名称或者函数指针来进行调用。而通过Block，源代码中就能够使用匿名函数。

”带有自动变量的匿名函数“这一概念并不仅指Block,它还用于其他许多程序语言中。
例如:

程序语言 Block名称
C+Blocks Block
Smalltalk Block
Ruby Block
LISP Lambda
C+Python Lambda
C++11 Lambda
JavaScript Anonymous function

Block 模式

Block语法

Block表达式语法与一般的C语言函数定义相比，仅有两点不同。

  1. 没有函数名
  2. 带有”^”

第一点没有函数名，因为是匿名函数。第二点不同是返回值类型前带有”^”(插入记号)记号，便于在大量是的使用中便于查找。

^ 返回值类型 参数列表 表达式

当然在使用中也可省略为:

^ 参数列表 表达式 /// 无返回值类型

^(int count){
return (count+1);
}

^ 表达式 ///省略返回值和参数列表

^{

printf(“Block\n”)

}

Block 类型变量

与使用函数指针的源代码对比可知，声明Block类型变量仅仅是将声明函数指针类型变量的”*“变为”^“。Block的类型变量则与C语言类型变量相同，可作为以下用途:

  • 自动变量
  • 函数参数
  • 静态变量
  • 静态全局变量
  • 全局变量

  • register

Block存储域

Block通过存储域，可分为以下几类:

  • _NSConcreteStackBlock ///表示该类的对象block设置在栈上
  • _NSConcreteGlobalBlock /// 表示该类的对象block与全局变量一样，设置在程序的数据区域(.data)中
  • _NSConcreteMallocBlock ///表示该类的对象block设置在malloc函数分配的内存块(即:堆)中。

Block的类

类 设置对象的存储域
_NSConcreteStackBlock 栈
_NSConcreteGlobalBlock 程序的数据区
_NSConcreteMallocBlock 堆

应用程序的内存分配

配置在全局变量的block，从变量作用域外也可以通过指针安全的使用。但设置在栈上的Block,如果其所属的变量作用域结束，该block就该废弃。 由于block变量也是在栈上，同样的，如果其所属的变量作用域结束，则该block变量也会被废弃。

当然，Block提供了copy将Block和__block变量从栈上复制到堆上的方法来解决这个问题。这样，即使Block语法记述的变量作用域结束，堆上的Block还可以继续存在。

_NSConcreteStackBlock 栈 从栈复制到堆
_NSConcreteGlobalBlock 程序的数据区 什么也不做
_NSConcreteMallocBlock 堆 引用计数增加

不管__block变量配置在栈上还是在堆上，都能够正确的访问变量 通过Block得复制,__block变量从栈复制到堆.此时也可同时访问栈上的__block变量和堆上的__block变量。

我们可以发现，在__main_block_impl_0 中捕获了自动变量array.在C语言结构体不能含有附有__strong修饰符的变量。因为编译器不知道应何时进行C语言结构体的初始化和废弃操作，不能很好的管理内存。

但是Objective-C的运行时库能够准确的把握Block从栈复制以及堆上上的Block别废弃的时机，因此Block用结构体中即使含有__strong修饰符或__weak修饰符的变量，也可以适当进行初始化和废弃。 为此需要使用在__main_block_desc_0结构体中增加的成员变量copy和dispose，以及作为指针赋值给该成员变量的__main_block_copy_0函数和__main_block_dispose_0函数。

_Block_object_assign 函数调用相当于retain实例方法的函数，将对象赋值在对象类型的结构体成员变量中。

_Block_object_dispose 函数调用想当于调用了release函数，用于释放赋值在对象类型的结构体成员变量中的对象。

调用copy函数和 dispose函数的时机

函数 调用时机
copy函数 栈上的Block复制到堆上
dispose函数 堆上的Block被废弃时

栈上的Block复制到堆上的时机:

  • 调用Block的copy实例方法
  • Block作为函数返回值返回
  • 将Block赋值给附有__strong修饰符id类型的类或Block类型成员变量
  • 在方法名中含有usingBlock的Cocoa框架方法或Grand Central Dispatch的API中传递Block时。

__block变量和对象

__block说明符可指定任何类型的自动变量，在ARC有效时，id 类型以及对象类型变量必定附加所有权修饰符，缺省为附有__strong修饰符的变量。

在Block中使用附有 __strong 修饰符的id类型或对象类型自动变量的情况下，当Block从栈复制到堆时，使用_Block_object_assign 函数，持有Block 截获的对象。当堆上的Block被废弃时，使用__Block_byref_id_object_dispose函数，释放Block截获的对象。

在Block中使用附有 __block 修饰符的id类型或对象类型自动变量的情况下会发生同样的情况，当__block变量从栈复制到堆时，使用_Block_object_assign 函数，持有赋值给__block变量的对象。当堆上的__block变量被废弃时，使用__Block_byref_id_object_dispose函数，释放Block截获的对象。

Block的循环引用

block会存在导致retain cycles的风险，如果发送者需要 retain block 但又不能确保引用在什么时候被赋值为 nil， 那么所有在 block 内对 self 的引用就会发生潜在的 retain 环.

解决方案:

1、使用__weak

2、在不需要的时候将block置为nil

亦可以参考系统NSOperation使用 block。

乍一看这似乎是一个retain环：self retain了queue，queue retain了operation，而operation retain了completion block，而completion blockretain了self。不过，在这里，将operation添加到queue时，会使operation在某个时机被执行，然后从queue中remove掉（如果没有被执行，就会有大问题了）。一旦queue移除了operation之后，retain环就被打破了。

copy/release

在ARC无效的时候，一般需要手动将Block从栈复制到堆，同时也需要释放复制的Block.

只要Block有一次复制并配置到堆上，就可通过retain实例方法持有,但是对于配置在栈上的Block调用retain实例方法则不起作用。

上诉源代码中，虽然对赋值给blk_on_stack 的栈上的Block调用了retain实例方法，但是实际上对此代码不起任何作用，因此还是推荐使用copy实例方法持有 Block.

由于Block是C语言的扩展，所以我们使用了_Block_copy和_Block_release函数管理Block的内存。

在ARC,无效的时候，我们可以使用__block来解决循环引用的问题，因为此时当Block从栈复制到堆时，若Block使用的变量为附有__block修饰符的id类型或者对象类型的自动变量，则不会retain.否则会执行retain.
